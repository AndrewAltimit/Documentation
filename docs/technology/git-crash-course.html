What is Git?

Imagine you‚Äôre writing a book. Every time you make major changes, you save a new copy: ‚ÄúMyBook_v1.docx‚Äù, ‚ÄúMyBook_v2_final.docx‚Äù, ‚ÄúMyBook_v2_FINAL_FINAL.docx‚Äù (we‚Äôve all been there).

Git is like having a magical filing cabinet that:

  • Saves every version automatically
  • Shows you exactly what changed between versions
  • Lets multiple people work on the same book without chaos
  • Can teleport you back to any previous version instantly

Why Should You Care?

Without Git, coding is like:

  • Walking a tightrope without a safety net
  • Writing a novel with no way to undo mistakes
  • Trying to collaborate via email attachments (nightmare!)

With Git, you can:

  • Experiment fearlessly - Break things! You can always go back
  • Collaborate smoothly - No more ‚Äúwhose version is newest?‚Äù
  • Track your progress - See how your code evolved over time

The Basic Workflow (Restaurant Kitchen Analogy)

Think of Git like a restaurant kitchen:

  1. Working Directory = Your cutting board
    • Where you actively prepare (edit) your files
  2. Staging Area = The pass (where finished dishes wait)
    • Where you place files that are ready to be saved
  3. Repository = The recipe book
    • Where all your saved versions live permanently

The Flow:

Edit files ‚Üí Stage them ‚Üí Commit (save) them
(Prep food) ‚Üí (Ready to serve) ‚Üí (Add to menu)

Essential Commands (Your Git Toolbox)

Setting Up

Daily Workflow

Time Travel

Core Concepts

The Three States

Git has three main states for your files:

  1. Modified - You've changed the file but haven't saved it to Git yet
  2. Staged - You've marked a modified file to go into your next commit
  3. Committed - The data is safely stored in your local database

Basic Workflow

1. Modify files in your working directory
2. Stage the changes you want to commit
3. Commit, which takes the staged changes and stores them permanently

Going Deeper

Understanding Git's Design

Git is more than just a version control system‚Äîit's a content-addressable filesystem with a VCS user interface. Here's what makes it special:

Snapshots, Not Differences

Unlike other VCS, Git thinks of its data as snapshots of a miniature filesystem. Every commit is a complete snapshot of all your files at that moment.

Nearly Every Operation Is Local

Most Git operations only need local files and resources. You have the entire history on your local disk, making most operations instantaneous.

Git Has Integrity

Everything in Git is checksummed using SHA-1 hash. It's impossible to change any file without Git knowing about it.
